什么是渲染流水线（Render Pipeline）？

Created2024-03-28|Updated2024-04-06
|Post Views:

流水线，又名管线，英文是pipeline。“管线”这一名称曾给我理解这一概念带来了很大障碍，“渲染管线”听起来不明觉厉。
不过随着对计算机组成原理、Unity Shader及技术美术知识的学习，我逐渐理解了Pipeline。本质上，流水线就是从完整地做完一件事变成分段做，将各个流程交给专门的工人来做，以此提高执行效率。
在《Unity Shader 入门精要》这本书中，作者用一个简单例子讲清楚了流水线的核心思想：

我们先来看一下真实生活中的流水线是什么。在工业上，流水线被广泛应用在装配线上。
我们来举一个例子。假设，老王有一个生产洋娃娃的工厂，一个洋娃娃的生产流程可以分为4个步骤：第1步，制作洋娃娃的躯干；第2步，缝上眼睛和嘴巴;第3步，添加头发；第4步：给洋娃娃进行最后的产品包装。
在流水线出现之前，只有在每个洋娃娃完成了所有这4个工序后才能开始制作下一个洋娃娃 如果说每个步骤需要的时间是1小时的话，那么每4个小时才能生产一个洋娃娃。
但后来人们发现了一个更加有效的方法，即使用流水线。老王把流水线引入工厂之后，工厂 发生了很大的变化。虽然制作一个洋娃娃仍然需要 4个步骤，但不需要从头到尾完成全部步骤，而是每个步骤由专人来完成，所有步骤并行进行。也就是说，当工序1完成了制作躯干的任务并把其交给工序2时，工序1又开始进行下一个洋娃娃的制作了。
使用流水线的好处在于可以提高单位时间的生产量。在洋娃娃的例子中，使用了流水线技术后每1个小时就可以生产一个洋娃娃。

作者还介绍到，流水线存在这样一个特性：

可以发现，流水线系统中决定最后生产速度的是最慢的工序所需的时间。例如，如果生产洋娃娃的第二道工序需要的是两个小时，其他工序仍然需要1个小时的话，那么平均每两个小时才能生产出一个洋娃娃。即工序2是性能的瓶颈(bottleneck)。
理想情况下，如果把一个非流水线系统分成n个流水线阶段，且每个阶段耗费时间相同的话会使整个系统得到n倍的速度提升。

那么将流水线放到渲染的语境下是怎样运作的呢？

渲染流水线的工作任务与阶段划分

工作任务是？

渲染流水线的工作任务在于由一个三维场景出发、生成(或者说渲染)一张二维图像。
换句话说，计算机需要从一系列的顶点数据、纹理等信息出发,把这些信息最终转换成一张人眼可以看到的图像。

谁来负责做工作？

这个工作通常是由CPU和GPU共同完成的。

概念阶段？

应用、几何、光栅化

渲染流水线的实现办法

1.输入装配阶段 -> 2.顶点着色器 -> 3.曲面细分阶段 -> 4.几何着色器 ->
5.剪裁阶段 -> 6.光栅化阶段 -> 7.像素着色器 -> 8.输出合并阶段

CPU到GPU

  1. 把数据加载到显存中
  2. *粗粒度剔除工作
  3. 设置渲染状态
  4. 调用DrawCall

GPU流水线

顶点着色器

主要处理顶点的各种空间变换，以及实现逐顶点光照。一个顶点着色器必须完成的任务是：
把顶点坐标从模型空间转换到齐次裁剪空间。先留心 “透视除法” 以及 “归一化的设备坐标 NDC” 的概念，数学实现按下不表。

坐标变换。顾名思义，就是对顶点的坐标(即位置)进行某种变换。
顶点着色器可以在这一步中改变顶点的位置，这在顶点动画中是非常有用的。
例如，我们可以通过改变顶点位置来模拟水面、布料等。
但需要注意的是,无论我们在顶点着色器中怎样改变顶点的位置,一个最基本的顶点着色器必须完成的一个工作是，

接着通常再由硬件做透视除法后，最终得到归一化的设备坐标(NormalizedDeviceCoordinates，NDC)。

说到空间变换，就需要了解渲染涉及的不同坐标空间：

  • 模型/物体/局部空间（Model/Object/Local Space, OS）
  • 世界空间（World Space, WS）
  • 观察空间（View Space, VS）
  • 裁剪空间（Clip Space, CS）
  • 屏幕空间（Screen Space）

在不同的空间做变换，通常是通过将坐标与矩阵做乘积的方式实现。在上述空间之间，存在三种最典型的变换：

  • 模型变换（Model Transform）
  • 观察变换（View Transform）
  • 投影变换（Projection Transform）

MVP变换，就是Model模型、View观察、Projection投影变换三个单词的缩写，
是图形学中将三维空间的物体呈现在二维屏幕上十分重要的三个变换。

对于模型空间与MVP变换，在这里只是简单提及，今后有时间再专门写一篇笔记梳理。

除了上述的不同空间之间的变换，顶点的变换往往还用来模拟布料、水流等。

顶点着色器也可以访问纹理贴图一类的数据.
只是与像素着色器中不同的是, 顶点着色器中只能用 sampleLevel方法采样, 即必须手动指定 mipmap 等级.(顺便提到 ddx, ddy 函数)

曲面细分着色器

在这一步可以对三角形进行细分, 以此来增加模型细节。

几何着色器

其主要特点是可以创建与销毁集合体, 还看过的例子是公告板技术,
在几何着色器中将一个点扩展为一个四边形并在上面贴上树木或者云来代替几何模型, 以此来降低性能消耗.

裁剪

在顶点着色器中，模型的坐标已经被转化为了齐次裁剪空间坐标，并做透视除法，得到了归一化的设备坐标（NDC）。
裁剪就是在这一基础上，将完全位于视野外的图元丢弃，将部分位于视野内的图元做裁剪，而完全位于视野内的则保留。

*屏幕映射

屏幕映射所做的工作是将NDC转化为屏幕空间坐标。
对x和y坐标做相应的变化，但是z坐标不做任何改变。这一过程又叫视口变换。

这里可能牵涉到OpenGL和DirectX的坐标系定义差异问题。

  • OpenGL的起点在左下角
  • DirectX的起点位于左上角

*三角形设置

三角形设置的阶段是为了获得图元对像素的覆盖情况，利用图元的顶点，计算出包围图元的边。

一个计算三角网格表示数据的过程就叫三角形设置。它的输出是为了给下一阶段做准备。

*三角形遍历

三角形遍历(Triangle Traversal)阶段将会检查每个像素是否被一个三角网格所覆盖。如果被覆盖的话，就会生成一个**片元(fragment)**。
而这样一个找到哪些像素被三角网格覆盖的过程就是三角形遍历，这个阶段也被称为扫描变换(ScanConversion)
三角形遍历阶段会根据上一个阶段的计算结果来判断一个三角网格覆盖了哪些像素，
并使用三角网格3个顶点的顶点信息对整个覆盖区域的像素进行插值。

背面剔除：在“光栅化阶段”的“三角形遍历”过程中进行，
就在决定哪些像素将被填充的前一步，这个步骤确定哪些三角形不会对最终图像作出贡献，并将它们剔除。

顶点属性插值：这个过程在“三角形遍历”和“像素着色器”之间进行，
确切地说可能是在“三角形遍历”的末尾，这是在像素级别上确定顶点属性如何在像素之间进行插值的阶段。

片元着色器 / 像素着色器（Fragment/Pixel Shader）

处理光照,阴影,特效等等, 绝大部分被眼睛感知到的画面效果都是在这里做的.

逐片元操作 / 输出合并阶段（在DirectX中）

修改颜色、深度缓冲、进行混合

决定每个片元的可见性，包含一系列测试：

  • 裁剪测试：如果一个点在裁剪区域外，我希望它彻底不渲染，也不要对缓冲区造成任何影响，所以优先级最高
  • Alpha测试：优先于模板测试，当我们希望用一个不规则的图片进行模板的裁剪的时候，应该使用Alpha测试先过滤掉那些透明的片段，我们只希望剩下的不透明片段来更新模板缓存
  • 模板测试：我希望对模板值进行一些更新，所以哪怕我被其它东西挡住，导致这个片段显示不出来，我也要把模板值更新进去
  • 深度测试：最后决定这个片段是否渲染（涉及Early-z和Z-Prepass技术）

如果一个片元通过了所有的测试，就需要把这个片元的颜色值和已经存储在颜色缓冲区中的颜色进行合并，或者说是混合（Blend）。
